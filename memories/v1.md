# Agent 使用规范

## 基本原则

- 对用户可见内容用中文；代码标识符、命令、技术名词保留原文。
- 技术表述准确客观；不夸大、不承诺未验证结果。
- **自主验证优先**：能自动验证就不推给用户。前端改完用 `playwright-cli` 自动验证，不要让用户"打开浏览器看看"。仅主观判断（审美/文案）或自动化无法覆盖（OAuth/支付）时才提示手动确认。

## 工作区与版本控制

- 保护用户/协作者已有改动；不改无关文件。
- 仅当用户明确说明、或经用户审批后，才可执行 `git commit`、`git push`、`git reset --hard`、`git clean` 等 VCS 写/破坏性操作。
- 提交前检查 status/diff；只提交本次任务相关文件。
- 仅当用户明确说明、或经用户审批后，才可切换/创建分支或 worktree。

## 代码与测试

- 少量代码不提前拆文件/建目录。
- 避免低价值包装；单次调用且语义弱的函数直接内联。
- 保留关键中间变量，方便断点调试。
- 测试优先覆盖业务行为、关键分支、边界、错误处理、对外契约。
- 不为覆盖率测试低价值实现细节，除非其是产品契约、审计要求或排障依赖。

## 日志

- 错误处理节点（`catch`、错误分支、降级）必须记日志，不得静默吞错只返回错误响应。
- 记错误带完整对象与堆栈（如 `logger.error({ err }, msg)`），禁止只 log `err.message`。

## 文档写作

- 更新项目文档（agents.md/CLAUDE.md/README 等，不含 CHANGELOG/ADR；/init 等生成式产出同样适用）时只保留核心信息：架构、约定、命令、入口点、约束、已知陷阱；不展开内部实现逻辑、数据流、调用链，除非该细节是产品契约、审计要求或排障依赖。
- 文档只描述当前事实，不写历史沿革、迁移来源、旧仓库位置等背景故事（沿革属 git log / CHANGELOG）；仅当省略会导致无法理解当前设计时才一句话带过。

## 日期与时间格式

未指定时，时间默认中国时区（+08:00，不标注），用 `YYYY-MM-DD HH:MM:SS` 及其前缀连续子集（24h、不跳级），如 `2026-06-29 23:05:59`。精度按任务取档（文档到日、日志到秒），允许变通。

- 人读字符串（展示/注释/文档）用本格式；机器解析字符串（API、`new Date('…')`）用 ISO8601（`2026-06-29T23:05:59`）；类构造用语言原生写法。
- 文件名带时间用 `2026-06-29_230509`，纯日期用 `2026-06-29`。
- 仅目标环境客观禁用某分隔符时可变通，不以主观偏好换格式族；环境注入的斜杠日期转横杠。

## 工具、包管理与命令

- 文件读取/修改/创建优先用当前 Agent 的内置工具（如 `Read`、`Edit`、`Write`、`Patch` 等，如存在）；内置工具不满足时再用 Bash。
- 看目录优先 `lsd --tree --depth 2`。
- `Glob`/`Grep` 禁止直接扫 `/` 或 `~`。
- Python 用 `uv`：运行 `uv run`，安装 `uv add`。
- JS/TS 用 `bun`。
- bun 仅作执行与包管理;业务代码不依赖 bun(引 node 包、测试用 vitest、禁 `bun:test`/`Bun.*`),dev/启动入口及 bun 专属项目可用 bun API。
- 运行 Python 时设 `PYTHONIOENCODING='utf-8'`。
- 一次性 Python：`uv run python -c "..."`
- 临时依赖：`uv run --with <pkg> python -c "..."`；多个依赖重复 `--with`。
- 反复调试的 Python 脚本放 `temp/code/*.py`，用 `uv run <file>`。

## 临时文件

- 临时/中间/调试产物默认放项目根目录 `temp/`，不要散落到源码或配置目录，也不默认用 `/tmp` 或系统临时目录。
- 用户明确指定项目外路径、或工具强制输出位置时，按上下文处理。

## 搜索优先级

- 网络搜索优先用 `zhipu-web-search`,回退 `WebSearch`;启动 subagent 时按需把本要求带入其 prompt。

## 技能与浏览器策略

| 场景 | 规则 |
|---|---|
| 明确的前端页面/组件/UI 实现或美化 | 先用 `frontend-design` |
| 全新页面、无设计稿、需视觉风格/设计系统/UX 决策 | 先 `ui-ux-pro-max`，再 `frontend-design` |
| 站点抓取/交互（数据、登录态、表单） | 优先 `opencli`；无适配器回退 `playwright-cli` |
| 通用浏览器自动化（前端验证、截图、任意 DOM） | 优先 `playwright-cli` → `playwright-mcp` → 提示用户安装 |

- `opencli` 优先：站点抓取/交互有现成适配器时先用 `opencli <site> <command>`，避免重复爬虫；用法细节见 `opencli-*` skills。
- `opencli browser <session>` 与 `playwright-cli open` 禁止不带 session 名裸调用（opencli 为位置参数，playwright 为 `-s`）。sessionId 命名规则：
- 默认：`uv run python -c "import uuid; print(uuid.uuid4().hex[:8])"` 生成，格式 `{task}-{uuid}`（如 `login-a1b2c3d4`），首次计算并记住，后续复用。
- 共享：多任务共用浏览器时用固定语义名（如 `pw-auth-flow`）。
- 特例：用户明确要求时遵从其指定。
- 不自行安装浏览器；禁用 `npx playwright install`、`google`、`puppeteer` 绕过。`opencli` 同此。

## CodeGraph

In repositories indexed by CodeGraph (a `.codegraph/` directory exists at the repo root), reach for it BEFORE grep/find or reading files when you need to understand or locate code:

- **MCP tools** (when available): `codegraph_explore` answers most code questions in one call — the relevant symbols' verbatim source plus the call paths between them. `codegraph_node` returns one symbol's source + callers, or reads a whole file with line numbers. If the tools are listed but deferred, load them by name via tool search.
- **Shell** (always works): `codegraph explore "<symbol names or question>"` and `codegraph node <symbol-or-file>` print the same output.

If there is no `.codegraph/` directory, skip CodeGraph entirely — indexing is the user's decision.
